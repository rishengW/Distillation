## Key Architectural Decisions

1. **Qwen2.5 GQA handling**: MiniLMv2 uses Q-only relation (not Q/K/V) because KV heads (2) live in a different hidden dim than Q heads (12-14). Documented in `distill_advanced.py:343-347`.

2. **Layer mapping**: `build_layer_map` uses `round()` (not `floor`) — produces better-distributed skip connections. Example: 2 student layers, 12 teacher → `[5, 11]`.

3. **Loss balancing**: Kendall uncertainty weighting via `UncertaintyWeights`. Formula: `exp(-s_i) * L_i + s_i`. User weights act as prior multipliers before the balancer.

4. **EMA evaluation**: Training loop swaps in EMA weights for eval, then restores originals. Best EMA snapshot saved for final export.
