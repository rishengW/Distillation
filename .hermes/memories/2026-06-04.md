### 3. GPT-2 Conv1D bug in arch_utils.py — FIXED

**Bug**: `extract_last_q` in `advanced/arch_utils.py` returned full QKV concatenation `[B,T,3H]` for GPT-2 instead of Q-only `[B,T,H]`. Root cause: GPT-2 uses `Conv1D` with weight shape `[H,3H]` (not `[3H,H]` like Linear), so the guard `weight.shape[0] != hidden_size` never triggered.

**Fix** (applied by kiro/opus-4.8):
```python
if hasattr(q_projector, "nf"):           # Detect Conv1D
    return q_projector(normed)[..., :hidden]  # Slice Q-only
```

## Lessons

1. **Skills > persona file for role-switching**: Permanent personality goes in persona file. Task-specific role transformation goes in a skill. Load on-demand, revert after task.
2. **Never trust memory of an API**: The reviewer skill explicitly forbids reviewing from memory — always `web_search` the latest docs.
3. **Conv1D vs Linear weight layout**: HuggingFace `Conv1D` stores weight as `(in_features, out_features)` = `[H, 3H]`, while `nn.Linear` stores `(out_features, in_features)` = `[3H, H]`. Shape-based guards must account for this.
4. **Hermes self-evolution**: Skills + memory + cron jobs = the agent gets better at YOUR tasks over time. Today's work created a reviewer persona that will improve every review.
